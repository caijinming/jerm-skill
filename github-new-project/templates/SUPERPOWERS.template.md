# SUPERPOWERS.md

## Purpose

本仓库采用 Superpowers 作为代理工作方法，而不是作为项目运行时依赖。

## Expected Workflow

1. `brainstorming`：澄清目标、边界和非目标
2. `writing-plans`：形成实现计划
3. `executing-plans` 或 `subagent-driven-development`：执行任务
4. `requesting-code-review`：对照计划和质量标准检查
5. `finishing-a-development-branch`：验证、收尾、提交

## Repository Convention

- OpenSpec 负责长期真相和变更记录
- Superpowers 负责工作方法
- `AGENTS.md` 负责仓库最低协作规则
