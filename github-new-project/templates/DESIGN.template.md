# DESIGN.md

## Purpose

这个文件用于记录产品与界面层的长期设计约束。

当前仓库还没有具体产品界面，因此这里只定义通用原则，后续随着真实需求补充。

## Product Tone

- 气质：`清晰、克制、专业`
- 体验目标：`低噪音、高可读、强调结构化信息`

## Visual Direction

- 不使用无意义的模板化视觉装饰
- 强调信息层级、状态清晰和可扫描性
- 配色、排版和动效都应服务于可读性

## Interaction

- 能直接反馈状态的交互，不依赖猜测
- hover、focus、disabled、error 状态必须明确
- 谨慎使用动画，避免打断任务流

## Documentation Rules

- 与设计相关的重要决策，优先沉淀到 OpenSpec
- 如果未来引入前端设计系统，再把 token、组件规则、页面模式补到这里

## Do Not

- 不要在没有产品目标时先写死 UI 风格
- 不要混用多个互不相关的视觉体系
- 不要把临时页面草稿当成长期设计规范
