# <PROJECT_NAME> OpenSpec Project

## Purpose

这个仓库在业务代码开始之前，先使用 OpenSpec 固化协作、变更和恢复上下文的方式。

## What Requires a Change

以下类型的变化必须先创建 OpenSpec change：

- 仓库结构的实质变化
- 协作流程的实质变化
- 代理工作方式的实质变化
- 设计规范的长期变化
- 技术栈引入与替换

## Current Baseline

当前仓库已建立这些基线能力：

- `change-management`
- `repository-bootstrap`

## Non-Goals

- 当前不定义具体业务能力
- 当前不绑定具体语言栈
- 当前不要求部署、测试或构建系统
